# 大片幅相機移軸互動模擬器 MVP

## 產品需求文件（PRD）

**產品名稱（暫定）**：View Camera Movement Simulator
**版本**：MVP v0.1
**平台**：Desktop Web Browser
**文件狀態**：Draft

---

## 1. 產品概述

本產品是一個以 Web 為基礎的大片幅相機移軸教學模擬器。使用者透過操作一部虛擬 4×5 大片幅相機的前組 movement，觀察相機幾何結構、3D 場景取景、地面玻璃畫面、焦平面及景深如何同步改變。

MVP 聚焦於三種最具代表性的 movement：

* **Rise**：在不抬高相機視軸的情況下，改變垂直取景範圍。
* **Tilt**：改變鏡頭平面與焦平面的關係，控制前後方向的清晰範圍。
* **Swing**：改變鏡頭平面與焦平面的關係，控制左右方向的清晰範圍。

產品目標不是複製某一特定品牌相機或鏡頭的全部光學特性，而是以準確、可視化、可互動的方式，讓使用者理解大片幅相機的基本幾何與操作邏輯。

---

## 2. 問題與機會

大片幅相機的 movement 學習門檻高，主要困難包括：

* 地面玻璃畫面倒轉及左右反轉，初學者難以即時判斷取景變化。
* Rise、tilt、swing 的結果難以只靠文字或影片理解。
* Scheimpflug principle、焦平面與景深的關係過於抽象。
* 實體大片幅器材成本高、操作慢，每次試錯均涉及底片、測光及沖洗成本。
* 一般數碼相機的 tilt-shift 效果展示，往往只呈現畫面變形，未能解釋真正前組與後組 movement 的幾何原理。

本產品透過即時互動，讓使用者能夠在低成本環境下反覆操作、比較結果及理解原理。

---

## 3. 產品目標

### 3.1 主要目標

1. 讓初學者理解 rise、tilt、swing 分別解決甚麼拍攝問題。
2. 讓使用者同時看到「操作」、「幾何關係」及「最終畫面」。
3. 讓使用者理解 tilt／swing 並非單純「增加景深」，而是改變焦平面方向。
4. 提供可完成的任務式練習，讓使用者從結果回推正確 movement。
5. 建立可擴充架構，為未來加入 rear movement、shift、底片操作及鏡頭資料庫作準備。

### 3.2 成功指標

| 指標                             |   MVP 目標 |
| ------------------------------ | -------: |
| 使用者完成三個基本 movement 教學關卡        |    ≥ 70% |
| 使用者能在無提示下完成至少一個關卡              |    ≥ 40% |
| 使用者在單次體驗中主動調整 movement 超過 10 次 |    ≥ 60% |
| 操作後地面玻璃及幾何圖更新延遲                |  < 100ms |
| Desktop WebGL 目標更新率            | ≥ 30 FPS |

## 4. 目標使用者

### 4.1 主要使用者

* 對大片幅攝影有興趣，但未曾實際使用 view camera 的攝影愛好者。
* 已使用數碼相機或中片幅相機，但不理解 tilt、swing、rise 原理的使用者。
* 攝影教師、工作坊導師及學生。
* 想在購買 4×5 相機前理解其操作方式的使用者。

### 4.2 使用者特徵

* 具基本攝影概念，例如光圈、焦距、景深、構圖。
* 未必理解鏡頭平面、底片平面、焦平面及 Scheimpflug principle。
* 主要使用桌面電腦或大型平板。
* 需要視覺化及反覆試驗，而非只閱讀技術文章。

## 5. MVP 範圍

### 5.1 包含功能

1. 2D 幾何示意圖
2. 3D 場景與虛擬相機模型
3. 地面玻璃即時預覽
4. 前組 rise、tilt、swing 即時調整
5. 手動對焦控制
6. 光圈與景深模擬
7. 焦平面模擬及可視化
8. 三個引導式學習關卡
9. 自由練習模式
10. 基本完成評分與操作回饋

### 5.2 MVP 不包含功能

* Rear rise、rear tilt、rear swing、rear shift。
* Front shift、fall。
* 特定品牌相機、鏡頭或精確機械行程資料庫。
* 真實鏡頭像差、色散、球面像差、散景形狀及微反差模擬。
* 光線追蹤級光學渲染。
* 完整底片裝載、暗片、快門上弦及沖洗流程。
* 帳戶系統、雲端存檔、社群分享。
* 手機優先介面。
* 自訂上傳 3D 場景。
* 多鏡頭、多片幅格式或 8×10 支援。

## 6. 核心產品設定

### 6.1 虛擬相機預設

MVP 使用一部抽象化的 4×5 view camera，不對應特定商業型號。

| 項目        | 預設值                  |
| --------- | -------------------- |
| 底片格式      | 4×5 inch             |
| 鏡頭焦距      | 150mm                |
| 初始光圈      | f/11                 |
| 可選光圈      | f/5.6、f/11、f/22、f/32 |
| 對焦方式      | 模擬皮腔延伸／focus rail 控制 |
| 前組 Rise   | 0–40mm               |
| 前組 Tilt   | -10° 至 +10°          |
| 前組 Swing  | -10° 至 +10°          |
| 後組位置      | 固定                   |
| 相機高度與拍攝位置 | 各教學關卡預設固定            |

### 6.2 操作原則

* 使用者只調整前組 rise、tilt、swing、focus 及 aperture。
* 相機位置、底片平面及後組預設固定，減少初學者混淆。
* 每次調整均同步更新 3D 場景、地面玻璃、焦平面及 2D 幾何圖。
* 所有 movement 均提供 reset control。
* Guided Mode 可限制可用控制項；Free Mode 開放全部 MVP 控制。

## 7. 使用者流程

### 7.1 首次使用流程

1. 使用者進入首頁。
2. 系統簡短介紹 rise、tilt、swing 三種 movement。
3. 使用者選擇「引導學習」或「自由操作」。
4. 若選擇引導學習，從 Rise 關卡開始。
5. 系統顯示任務目標、操作提示及成功條件。
6. 使用者操作 movement。
7. 系統即時更新畫面及幾何圖。
8. 使用者達成目標後獲得結果、解說及下一關入口。

### 7.2 自由操作流程

1. 使用者選擇場景。
2. 系統載入預設相機位置及鏡頭設定。
3. 使用者調整 rise、tilt、swing、focus 及 aperture。
4. 使用者於地面玻璃、3D 場景及幾何圖之間比較結果。
5. 使用者可重設相機或切換另一場景。

## 8. 功能需求

## FR-01：3D 場景與相機模型

系統必須提供一個可視化 3D 場景，讓使用者看到虛擬相機、鏡頭前組、底片平面、被攝物及相機視軸。

### 要求

* 顯示 4×5 相機基本結構：

  * 前組。
  * 後組。
  * 鏡頭板。
  * 皮腔。
  * 底片平面。
  * 光軸。
* 顯示前組 rise、tilt、swing 的即時位置變化。
* 顯示被攝場景與相機相對位置。
* 支援旋轉、縮放及重設 3D 外觀視角。
* 不允許使用者在 MVP 中移動整部相機的位置。

### 驗收條件

* 使用者調整任一 movement 後，3D 模型於 100ms 內反映變化。
* 前組 tilt 與 swing 必須呈現明確旋轉方向。
* rise 必須呈現鏡頭中心在垂直方向的平移。

---

## FR-02：地面玻璃預覽

系統必須模擬大片幅相機的地面玻璃畫面，作為使用者理解構圖、對焦及景深變化的主要介面。

### 要求

* 顯示由虛擬鏡頭投影至 4×5 底片平面的畫面。
* 預設使用大片幅地面玻璃方向：

  * 上下倒轉。
  * 左右反轉。
* 提供「方向輔助」開關：

  * 開啟時把畫面轉正。
  * 僅作學習輔助，不影響實際模擬計算。
* 顯示基本構圖格線及中心線。
* 顯示目前光圈、focus、rise、tilt、swing 數值。
* 支援地面玻璃局部放大模式。
* 允許開啟「焦點輔助 overlay」，以非單一顏色方式提示清晰程度。

### 驗收條件

* movement 改變後，地面玻璃構圖及清晰範圍必須即時更新。
* 地面玻璃在關閉方向輔助時，必須保留倒像及左右反轉。
* focus 或 aperture 改變後，模糊程度必須可辨識。

---

## FR-03：2D 幾何示意圖

系統必須使用 2D 幾何圖，將難以從地面玻璃直接理解的相機關係可視化。

### 要求

系統提供兩個可切換的幾何視圖：

#### 側視圖

適用於 rise、tilt、focus。

必須顯示：

* 底片平面。
* 鏡頭平面。
* 光軸。
* 鏡頭中心。
* 被攝主體的代表平面。
* 焦平面。
* 景深近端與遠端的近似範圍。
* tilt 前後鏡頭平面與焦平面的變化。

#### 俯視圖

適用於 swing、focus。

必須顯示：

* 底片平面。
* 鏡頭平面。
* 光軸。
* 被攝物的水平分布。
* 焦平面。
* swing 前後焦平面方向變化。

### 驗收條件

* 當使用者調整 tilt 時，側視圖中的鏡頭平面及焦平面必須同步變化。
* 當使用者調整 swing 時，俯視圖中的鏡頭平面及焦平面必須同步變化。
* 當使用者調整 aperture 時，景深範圍必須改變，但焦平面中心方向不應錯誤偏移。
* 幾何圖不得只作裝飾用途，必須由與畫面預覽相同的相機狀態驅動。

---

## FR-04：Rise 模擬

系統必須模擬前組 rise 對垂直取景範圍的影響。

### 要求

* Rise 控制範圍為 0–40mm。
* Rise 以滑桿、數值輸入及 3D 拖曳控制至少其中兩種方式實現。
* 調整 rise 時：

  * 地面玻璃可見畫面向上移動。
  * 3D 模型中的鏡頭板垂直移動。
  * 側視幾何圖顯示鏡頭中心位置改變。
* 於建築場景中，系統應讓使用者看見：

  * 保持相機水平時，rise 可收入更多建築上方內容。
  * 不需要抬高相機視軸亦可包含建築頂部。

### 驗收條件

* Rise 由 0 調整至最大值時，地面玻璃必須可見明顯更多場景上方內容。
* rise 不應直接改變鏡頭 tilt 或 swing 角度。
* Reset 後必須回到 0mm。

---

## FR-05：Tilt 模擬

系統必須模擬前組 tilt 對焦平面方向及前後景清晰範圍的影響。

### 要求

* Tilt 控制範圍為 -10° 至 +10°。
* 使用者可調整 focus 與 tilt。
* 系統依據鏡頭平面、底片平面、焦距及對焦位置，計算近似焦平面。
* 系統必須在地面玻璃、3D 場景及側視幾何圖中，同步顯示 tilt 結果。
* 系統必須讓使用者看見：

  * 沒有 tilt 時，桌面前景及遠端物件未必同時清晰。
  * 適量 tilt 後，焦平面可沿桌面或地面方向延伸。
  * 過量 tilt 可令重要主體失焦。
  * aperture 影響景深範圍，但不等同於 tilt 的焦平面控制。

### 驗收條件

* Tilt 調整後，焦平面方向必須有可見變化。
* f/5.6 與 f/32 必須呈現不同的清晰範圍。
* 改變 aperture 時，焦平面中心方向不應改變。
* 使用者能在桌面場景中，透過 tilt 加上適當 focus 令指定近、中、遠目標進入可接受清晰範圍。

---

## FR-06：Swing 模擬

系統必須模擬前組 swing 對左右方向焦平面及清晰範圍的影響。

### 要求

* Swing 控制範圍為 -10° 至 +10°。
* 系統提供左右延伸或斜向排列的被攝主體。
* 系統必須在地面玻璃、3D 場景及俯視幾何圖中，同步顯示 swing 結果。
* 系統必須讓使用者看見：

  * 無 swing 時，斜向排列的主體未必同時清晰。
  * 適量 swing 後，焦平面可沿主體方向旋轉。
  * 過量 swing 可令原本清晰的主體失焦。

### 驗收條件

* 調整 swing 時，俯視幾何圖中的焦平面方向必須改變。
* 使用者能於斜向書架或走廊場景中，令指定物件同時進入可接受清晰範圍。
* Swing 不應改變 vertical rise 數值。

---

## FR-07：景深與焦平面模擬

系統必須提供教學級景深模擬，讓使用者理解「焦平面」與「景深」是相關但不同的概念。

### 要求

* 以簡化薄透鏡模型計算 focus 與離焦程度。
* 使用 circle of confusion 概念估算可接受清晰範圍。
* 根據 aperture、物體與焦平面的距離，套用可見模糊效果。
* 提供景深輔助模式：

  * 清晰區。
  * 接近可接受清晰區。
  * 明顯失焦區。
* 不必模擬真實鏡頭像差、散景光斑或特定鏡頭風格。
* 畫面須標示「教學級光學模擬」，避免使用者誤解為真實鏡頭測試工具。

### 驗收條件

* aperture 變大時，景深必須縮小。
* aperture 收細時，景深必須增加。
* focus 改變時，最佳清晰區必須移動。
* tilt／swing 改變時，最佳清晰區方向必須改變。

---

## FR-08：引導式學習關卡

MVP 必須包含至少三個關卡，每關集中教授一種 movement。

### 關卡一：建築頂部與 Rise

**場景**：固定位置拍攝高樓立面。
**任務**：在不抬高相機視軸的情況下，將建築頂部完整收入畫面。
**主要學習目標**：rise 改變取景範圍。
**可用控制**：rise、focus、aperture。
**成功條件**：

* 建築頂部完整出現在地面玻璃畫面內。
* 主要建築立面未被裁切。
* rise 數值在合理範圍內。

### 關卡二：桌面靜物與 Tilt

**場景**：由近至遠延伸的桌面，擺放多個標示物件。
**任務**：以不大於 f/22 的光圈，令近、中、遠三個目標同時可接受清晰。
**主要學習目標**：tilt 改變前後焦平面。
**可用控制**：tilt、focus、aperture。
**成功條件**：

* 三個目標均進入可接受清晰範圍。
* aperture 不可小於 f/22。
* tilt 不可超過預設合理範圍。

### 關卡三：斜向主體與 Swing

**場景**：斜向延伸的書架、欄杆或走廊物件。
**任務**：令前景、中景、遠景的指定標記同時清晰。
**主要學習目標**：swing 改變左右方向焦平面。
**可用控制**：swing、focus、aperture。
**成功條件**：

* 三個標記均進入可接受清晰範圍。
* 使用者必須調整 swing，不能只依賴最大景深。
* aperture 不可小於 f/22。

---

## FR-09：即時回饋與結果頁

系統必須在使用者完成或接近完成目標時，提供簡短且具體的教學回饋。

### 回饋原則

* 描述使用者做了甚麼。
* 解釋畫面為何會產生該結果。
* 指出更合理或更有效的操作方向。
* 不以單純「正確／錯誤」取代說明。

### 回饋例子

> 你以 rise 收入了建築頂部，而沒有改變相機視軸，因此建築線條維持自然。

> 你嘗試單靠收細光圈增加清晰範圍，但桌面前後距離過大。調整 tilt 可令焦平面更接近桌面方向。

> 你的 swing 角度過大，焦平面已偏離遠端主體。可先回到較小角度，再重新 focus。

### 結果頁必須顯示

* 是否完成任務。
* 使用的 movement 數值。
* 使用的 aperture。
* 最終 focus 設定。
* 成功原因或失敗原因。
* 重試按鈕。
* 下一關按鈕或自由練習入口。

## 10. 非功能需求

### 10.1 平台與瀏覽器

* 優先支援 Desktop Chrome、Edge、Safari。
* 最低建議顯示寬度為 1280px。
* 不保證手機完整操作體驗。
* 偵測 WebGL 不支援時，顯示明確提示。

### 10.2 效能

* 控制輸入至視覺更新目標少於 100ms。
* 正常 desktop 裝置下，3D 場景目標維持至少 30 FPS。
* 場景載入後不得因連續拖動 slider 而顯著卡頓。
* MVP 場景及模型須採低至中等複雜度，優先保留互動流暢度。

### 10.3 可及性

* 所有滑桿可使用鍵盤調整。
* 關鍵狀態不可只依賴顏色表示。
* 控制項需有文字 label、數值及 tooltip。
* 提供關閉動畫或降低動畫效果的選項。
* 重要教學內容須以文字表達，不只依賴圖像。

### 10.4 私隱

* MVP 不要求登入。
* 不收集個人身份資料。
* 若加入行為分析，只記錄匿名操作事件，例如關卡開始、重試、完成及 movement 調整次數。

---

## 11. 技術實作原則

### 11.1 前端架構建議

* TypeScript。
* React。
* Three.js 或 React Three Fiber。
* SVG 或 Canvas 用於 2D 幾何圖。
* WebGL 用於 3D 場景及地面玻璃即時渲染。
* Client-side state management，用於同步相機狀態、場景狀態及 UI。

### 11.2 模擬模型原則

MVP 採用簡化的傾斜薄透鏡模型，而非完整 ray tracing。

核心相機狀態包括：

```ts
type CameraState = {
  focalLengthMm: number;
  aperture: number;
  focusDistance: number;

  frontRiseMm: number;
  frontTiltDeg: number;
  frontSwingDeg: number;

  lensPlane: Transform;
  filmPlane: Transform;

  scenePreset: string;
};
```

系統須由同一份 `CameraState` 驅動：

* 3D 相機模型。
* 地面玻璃預覽。
* 2D 側視圖。
* 2D 俯視圖。
* 焦平面。
* 景深模糊效果。
* 關卡完成判定。

不得為不同畫面使用互不一致的近似計算。

## 12. 資料與內容需求

### 12.1 場景資產

MVP 至少需要三個靜態 3D 場景：

| 場景      | 學習 movement | 必要物件            |
| ------- | ----------- | --------------- |
| 建築立面    | Rise        | 高樓、天空、地面、窗格或垂直線 |
| 桌面靜物    | Tilt        | 延伸桌面、近中遠標記物件    |
| 斜向書架／走廊 | Swing       | 斜向延伸主體、三個清晰度目標  |

### 12.2 教學內容

每個關卡需要：

* 關卡目標。
* 簡短背景。
* 初始相機設定。
* 可用 controls。
* 成功條件。
* 提示文字。
* 完成後說明。
* 常見錯誤回饋。

## 13. 分析事件

MVP 建議記錄以下匿名事件：

| 事件                                 | 說明                    |
| ---------------------------------- | --------------------- |
| `scene_started`                    | 使用者開始關卡或自由模式          |
| `movement_changed`                 | 使用者調整 rise、tilt、swing |
| `focus_changed`                    | 使用者調整 focus           |
| `aperture_changed`                 | 使用者調整 aperture        |
| `hint_opened`                      | 使用者查看提示               |
| `ground_glass_orientation_toggled` | 使用者切換方向輔助             |
| `scene_completed`                  | 使用者完成關卡               |
| `scene_restarted`                  | 使用者重試關卡               |

事件不得包含姓名、電郵、IP 或可識別個人資料。

## 14. 主要風險與處理方式

| 風險                | 影響 | 緩解方式                         |
| ----------------- | -- | ---------------------------- |
| 光學簡化導致教學錯誤        | 高  | 清楚標示為教學級模擬；以正確幾何關係優先於寫實渲染    |
| UI controls 過多    | 高  | Guided Mode 每關只開放必要 controls |
| 使用者只靠最大光圈縮小景深問題   | 中  | 關卡限制最大可用光圈，例如不小於 f/22        |
| 地面玻璃倒像令初學者放棄      | 中  | 提供可切換方向輔助，但預設保留真實模式          |
| 景深效果效能不足          | 中  | 使用螢幕空間模糊及簡化 CoC 計算           |
| 3D 畫面過於寫實導致開發範圍膨脹 | 中  | 採風格化、低至中複雜度場景，優先教學清晰度        |

## 15. MVP 驗收清單

產品可進入 MVP 測試的最低條件：

* [ ] 使用者可在瀏覽器載入至少三個 3D 教學場景。
* [ ] 使用者可調整前組 rise、tilt、swing。
* [ ] 每次 movement 改變均同步更新 3D 相機、地面玻璃及 2D 幾何圖。
* [ ] 地面玻璃具倒像及左右反轉模式。
* [ ] 地面玻璃可呈現可辨識的 focus 與景深差異。
* [ ] Tilt 可改變側視圖中的焦平面方向。
* [ ] Swing 可改變俯視圖中的焦平面方向。
* [ ] Rise 可改變地面玻璃中的垂直取景範圍。
* [ ] 每一關均有可量化成功條件。
* [ ] 關卡完成後可顯示具體操作回饋。
* [ ] 系統在一般 desktop 瀏覽器保持可操作流暢度。

## 17. MVP 定義

本 MVP 成功的標準不是產生完全等同真實大片幅鏡頭的相片，而是讓使用者能夠透過操作與結果之間的因果關係，清楚回答以下問題：

* 為甚麼拍建築時要用 rise，而不是直接把相機向上傾？
* 為甚麼 tilt 可以令桌面前後物件同時清晰？
* 為甚麼 swing 對斜向排列的主體有用？
* 焦平面與景深有甚麼分別？
* 為甚麼收細光圈不能完全取代正確的 movement？

只要系統能準確、直觀及可重複地讓使用者理解以上問題，即符合 MVP 的核心目標。
